Data Collection Monitoring Tool Guide

This tool helps users monitor trials for their Minimum Viable Products (MVPs). It guides users through a series of steps to crosscheck submissions and track progress in near real-time. Additionally, it facilitates seamless communication with relevant extension agents in the field.

I. Define the trial conditions

On the left-side panel of the tool, the user is initially provided with a set of options to define the trial conditions.

Define the Stage

The user will need to select the stage for which the data is being collected. This could be the validation, piloting, or NOT trials.

Define the Experiment

The user will need to select the trial experiment.

Define the Date

The user will need to specify the trial period for which to monitor the submissions. The default period initially selected is the earliest date of submission and the current date. These vary for different use cases.

Select the Enumerator

This filter provides a list of all enumerators' identification numbers registered for the trial, in ascending order. The initial selection shows details for all the enumerators. The user can select specific enumerators to view their progress. Multiple selections are allowed in this case.

Select the Household

This filter provides a list of all registered households' identification numbers for the trial, in ascending order. The initial selection shows details for all the households. The user can select specific households to view their progress. Multiple selections are allowed in this case.

II. Summary

The Summary page displays the overview of the submissions for the selected trial. The general use case information; the use case name, and the country where the trials are conducted are shown at the top.

The Total Submissions: Displays the overall number of unique submissions for the trial, without any duplicates.

The Trials by Location: Displays a map pinpointing the locations of various submissions.

The Trend of Submission: This shows a trend line graph and visually represents data points over time to illustrate how submissions are progressing.

Summary of Complete Submissions: The first table provides the total number of completed submissions from all enumerators, aggregated by event. The second table provides the total number of completed submissions from each enumerator, aggregated by event. The user can opt to download the results for offline or field use.

III. Enumerators

The Enumerators page displays a timeline of submission events for each enumerator and household. The dates indicate when each specific event was submitted. On ENID, HHID hover, the user can access the enumerator and household name and contact information respectively.

Different colors are used to indicate the timeliness of data submitted based on specific periods required between the events, as provided by the agronomist:

  • Green: The submission was completed within the required timeframe.
  • Orange: The submission is missing but is still within the allowed submission period.
  • Red: The submission is overdue and has exceeded the required period for completion.
  • Purple: The event is scheduled for the future, and the submission period has not yet begun.
  • White: If submission for other events was completed but Event 1 (usually planting) is not yet submitted.

IV. Issues

The Issues page displays various submissions that have been flagged as unusual and require attention. Issues may include:

  • ENID or HHID Mismatches: Instances where the ENID or HHID do not adhere to the expected naming conventions.
  • Event Order Issues: Cases where submission events do not follow the expected sequence or where events are skipped.

V. Data Preview

This page displays a table with clean, organized data, allowing you to review the information before downloading. You can download the data as a CSV file for further analysis or record-keeping. The table includes all relevant fields and entries, ensuring that you have access to accurate and well-structured data.

VI. Glossary

The Glossary page provides definitions and explanations for key terms and concepts used throughout the system. It serves as a reference to help users understand specialized vocabulary and acronyms. This page is organized for easy navigation, with terms listed by category, ensuring quick access to essential information.
